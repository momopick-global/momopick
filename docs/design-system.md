# Design System — 모모픽 (1차 오픈)

> **목적**: PC·태블릿·모바일 웹에서 **일관된 브랜드**와 **접근 가능한 UI**를 유지.  
> Figma에서 **컴포넌트 라이브러리**로 옮길 때 이 문서를 단일 소스로 삼습니다.

---

## 1. 브랜드 원칙

| 원칙 | 설명 |
|------|------|
| **가볍게 시작** | 1분 퀴즈는 진입 장벽 최소 — 질문·선택지는 짧게. |
| **결과가 보상** | 결과 화면은 시각적 만족(타이포·일러스트·공유 카드). |
| **신뢰** | 심리·연애 카피는 과장·의학적 단정 금지(법무 가이드 연동). |

레퍼런스(푸망, 소울블렌드, 청월당)는 **정보 밀도·카테고리 허브·공유 UX**만 벤치마크하고, 색·타이포는 모모픽 고유성 유지.

---

## 2. 반응형 브레이크포인트

| 토큰 | 범위 | 용도 |
|------|------|------|
| `sm` | 0–479px | 소형 폰 |
| `md` | 480–767px | 폰 가로·큰 폰 |
| `lg` | 768–1023px | 태블릿 |
| `xl` | 1024px+ | PC |

**Figma 프레임 권장**: 390×844, 768×1024, 1280×800.

**모바일 퍼스트**: 기본 스타일은 `sm`, `lg` 이상에서 그리드·사이드 여백 확장.

---

## 3. 컬러 (CSS 변수 예시)

현재 정적 페이지 톤(**다크 베이스 + 퍼플 액센트**)을 기준으로 토큰화:

| 토큰 | 용도 | 참고 값(다크) |
|------|------|----------------|
| `--color-bg` | 페이지 배경 | `#0b0f17` |
| `--color-surface` | 카드·패널 | `rgba(255,255,255,.06)` |
| `--color-text` | 본문 | `rgba(255,255,255,.92)` |
| `--color-muted` | 보조 텍스트 | `rgba(255,255,255,.68)` |
| `--color-border` | 구분선 | `rgba(255,255,255,.14)` |
| `--color-accent` | CTA·강조 | `#5A67F2` |
| `--color-accent-2` | 보조 강조(연애 등) | `#ff6b9a` 계열 |

**라이트 모드**: `prefers-color-scheme: light` 대응 시 동일 토큰에 라이트 팔레트 매핑(기존 `index.html` 패턴 재사용).

**대비**: 본문 vs 배경 **WCAG AA** 이상(4.5:1) 목표. 액센트 위 흰 글씨는 버튼 크기·굵기로 보완.

---

## 4. 타이포그래피

| 스타일 | 용도 | 권장 |
|--------|------|------|
| **Display** | 히어로·결과 타이틀 | 시스템 폰트 스택 또는 브랜드 웹폰트 1종만 |
| **Title** | 페이지 제목 | 1.25–1.5rem (모바일) |
| **Body** | 본문 | 16px 이상(모바일 가독성) |
| **Caption** | 법적 고지·메타 | 12–13px, `--color-muted` |

**줄간격**: 본문 1.5–1.65.  
**한글**: `Apple SD Gothic Neo`, `Noto Sans KR` 등 시스템 스택 유지 가능.

---

## 5. 스페이싱 & 그리드

| 토큰 | 값 |
|------|-----|
| `--space-1` | 4px |
| `--space-2` | 8px |
| `--space-3` | 12px |
| `--space-4` | 16px |
| `--space-6` | 24px |
| `--space-8` | 32px |

**페이지 패딩**: 모바일 `16–24px`, 태블릿 이상 `24–32px`.  
**최대 콘텐츠 너비**: 읽기 영역 `min(720px, 100%)` (기존 카드 폭과 정렬).

---

## 6. 컴포넌트 (1차 필수 세트)

| 컴포넌트 | 상태 | 설명 |
|----------|------|------|
| **Button** | default / hover / disabled / loading | 주 CTA 1개만 강조 색 |
| **Card** | 테스트 카드(썸네일·제목·메타) | 그리드: 모바일 1열, 태블릿 2열, PC 3열 |
| **Chip / Tag** | 태그·카테고리 | 둥근 pill |
| **Progress** | 퀴즈 진행 바 | 스텝 수 노출 |
| **Question block** | 질문 + 옵션 리스트 | 터치 영역 최소 44×44pt |
| **Result hero** | 결과 타이틀 + 일러스트 영역 | 공유용 비율(1.91:1 등) 고려 |
| **Nav** | 상단 바 + 햄버거(모바일) | 고정 시 safe-area |
| **Footer** | 약관·공지·SNS | 모든 공개 페이지 |

---

## 7. 아이콘·일러스트

- **아이콘**: 단일 세트(SVG), 스트로크 두께 통일.
- **이모지**: 카테고리 라벨(💗 등)은 **접근성 대체 텍스트** 제공(`aria-label`).

---

## 8. 모션

- 전환: **150–250ms**, `ease-out`.
- 과한 자동 재생 애니메이션 금지(멀미·배터리).
- `prefers-reduced-motion: reduce` 시 애니메이션 축소.

---

## 9. 접근성

- 모든 이미지 `alt`.
- 포커스 링 visible (`:focus-visible`).
- 퀴즈 옵션은 `button` 또는 `role="radio"` + 키보드 조작.
- 언어 속성: `<html lang="ko">` / 영문 페이지는 `lang="en"`.

---

## 10. SEO·소셜(디자인 연동)

- **OG 이미지**: 1200×630 권장. 결과별 동적 이미지는 2차.
- **파비콘**: 다크/라이트 대응(선택).

---

## 11. 콘텐츠 가이드 (카피 톤)

| 카테고리 | 톤 |
|----------|-----|
| 연애 | 따뜻하지만 단정 금지(“당신은 반드시 ~”) |
| 심리·MBTI | 오락용 명시, 임상 오인 방지 |
| 재미 | 밈·가벼운 유머 허용, 혐오·차별 금지 |

---

## 12. Figma 작업 순서 제안

1. 컬러·타이포 토큰 페이지  
2. 그리드·브레이크포인트 프레임  
3. 핵심 컴포넌트(버튼·카드·퀴즈 블록)  
4. 템플릿: 허브 / 퀴즈 1페이지 / 결과 / 블로그 글
