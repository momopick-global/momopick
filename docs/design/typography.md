# Typography — 모모픽

> CSS 토큰화는 거의 안 되어 있음. 기존 `ko-home.css`의 패턴을 따라 일관성 유지.

## 폰트 스택

```css
font-family: -apple-system, BlinkMacSystemFont, "Apple SD Gothic Neo",
  "Noto Sans KR", "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

시스템 폰트 우선 → 빠른 로드, 한글 가독성 보장.
별도 웹폰트 미사용 (성능·로딩 시간 우선).

## 크기 가이드

| 용도 | 권장 크기 | 굵기 | 비고 |
|---|---|---|---|
| 페이지 메인 타이틀 (H1) | 22~28px | 700 | 모바일 22, 데스크톱 28 |
| 섹션 타이틀 (H2) | 18~22px | 700 | |
| 카드 제목 | 16~17px | 600~700 | 짧게 |
| 본문 | 15~16px | 400 | 모바일에서도 16 권장 |
| 메타 / 캡션 | 12~13px | 400 | `--muted` 색 |
| 버튼 | 14~15px | 600 | |
| 결과 hero 타이틀 | 24~32px | 800 | 굵게, 결과는 보상감 |

## 줄간격 (line-height)

| 용도 | 값 |
|---|---|
| 본문 | 1.55~1.65 |
| 짧은 제목 | 1.2~1.35 |
| 한 줄 라벨/캡션 | 1 |

## 자간 (letter-spacing)

| 용도 | 값 |
|---|---|
| 큰 타이틀 (24px+) | `-0.01em` 또는 `-0.02em` (약간 조밀) |
| 본문 | 기본 (0) |

## 한글 줄바꿈

- `word-break: keep-all` — 한글 단어 중간 끊김 방지
- `overflow-wrap: anywhere` — 긴 영문 URL 줄바꿈
- 짧은 카드 제목은 `text-wrap: balance` (지원 브라우저에서)

## 모바일 가독성 주의

| 안티 패턴 | 권장 |
|---|---|
| 본문 12px | 본문은 최소 14, 권장 16 |
| 한 줄에 너무 긴 텍스트 | max-width로 줄 길이 제한 (45~70 char) |
| 굵은 글씨로만 강조 | 색 + 굵기 조합 |
| 카카오 인앱 브라우저에서 깨짐 | 큰 폰트 사이즈 (rem) 권장 (인앱은 base size 조정 가능) |

## 카카오 인앱 브라우저 글자 크기 이슈

카카오톡 인앱 브라우저는 일부 사용자가 OS 글자 크기 설정 또는 인앱 자체 줌으로 글자가 다르게 보일 수 있음. 모모픽은 `InAppBrowserGuide` 모달로 외부 브라우저 사용을 권장하지만, 그래도 인앱에서 깨지지 않게:

- 절대 크기(`px`) vs 상대 크기(`rem`) 중 일관되게 사용
- `font-size: 100%` 강제 X (사용자 설정 존중)
- 줄간격은 충분히 (1.5+)

## 강조 패턴

| 강조 종류 | 패턴 |
|---|---|
| 핵심 키워드 | `<strong>` + 색 (`--accent` 또는 카테고리) |
| 본문 중 링크 | `--accent` 색 + underline (호버 시 어둡게) |
| 인용 / 노트 | 카드형 박스 + `--muted` 텍스트 |

## 예시 - 결과 페이지

```html
<h1 class="result-title">  <!-- 28~32px / 800 -->
  나의 썸 패턴은 <span class="accent">밀당형</span> 🎭
</h1>
<p class="result-tagline">  <!-- 16px / 500 / --muted -->
  적당한 밀당이 아니라 타이밍을 놓치는 타입
</p>
<div class="result-body">   <!-- 15~16px / 400 / line-height 1.65 -->
  ...
</div>
```

## 새 텍스트 스타일 추가 시

1. 위 표에 매핑되는 용도인지 검토
2. 새로운 크기/굵기 추가는 정말 필요한 경우만
3. CSS 토큰화 추천 (`--text-h1`, `--text-body` 등) — 현재는 미적용 상태
