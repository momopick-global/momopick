# Blog Writing Guide — 블로그 작성

## 기본 구조

`src/content/blog/koSamplePosts.ts`의 객체 배열에 추가.

```ts
{
  id: "<slug>",                  // URL 슬러그로 사용 (영문 소문자 + 하이픈)
  date: "2026.05.25",            // 표시용
  dateTime: "2026-05-25",        // <time datetime=...> 용
  title: "✨ 글 제목",
  excerpt: "한 줄 요약 (60~120자)",
  body: "본문 (\\n\\n으로 문단 구분)",
  category: "love" | "personality" | "tarot" | "fun" | "story",
  tag: "가이드" | "에디터노트" | "업데이트" | "팁",  // 선택
  image: "/images/blog/<...>.webp",                  // 선택
  imageAlt: "이미지 설명",                            // 선택
}
```

## 본문 길이

| 길이 | AdSense 평가 |
|---|---|
| ~300자 | thin content 위험 |
| 600~1,000자 | 양호 |
| 1,500자+ | 풍부함 |

**AdSense 심사 통과를 위해 600자 이상 권장.** 현재 7개 글 중 5개가 220~320자 → 확장 또는 임시 제거 필요.

## 본문 구성 권장

1. **도입** (100~200자): 독자가 공감할 상황·문제 제시
2. **본문 1** (200~400자): 핵심 인사이트 1
3. **본문 2** (200~400자): 핵심 인사이트 2 또는 다른 측면
4. **마무리** (100~200자): 정리 + (선택) 모모픽 테스트 추천 링크

## 제목 작성

- 한국어 + 이모지 1개 권장
- 검색·공유에서 클릭하고 싶은 톤
- 너무 자극적이지 않게

| 좋은 예 | 별로 |
|---|---|
| 🔮 타로 카드란 무엇인가? | 타로카드 |
| 📌 썸이 항상 애매하게 끝나는 이유 | 썸 이별 |
| MBTI·성향 퀴즈, 진단이 아니라는 걸 기억해 주세요 | MBTI 정확함? |

## SEO 메타

페이지 (`src/app/ko/blog/[slug]/page.tsx`)가 자동으로 `metadata` 생성:
- title: `<post.title> | 모모픽 블로그`
- description: `excerpt` (160자 이내로 trim)
- canonical: `https://momopick.com/ko/blog/<slug>/`
- OG image: `post.image` (있으면 절대 URL로) 또는 `main-og.webp`

## H2 / H3 구조

본문 안에 명시적 마크다운/HTML이 없으면 `<p>` 위주로 렌더링. 헤딩 필요하면:
- 본문 안에 `## H2 제목` 같은 마크다운 사용은 현재 미지원
- 강조 본문 시작 줄에 `<strong>제목</strong>` 또는 컴포넌트 추가 필요

향후 MDX 도입 시 H2/H3 자유롭게.

## 내부 링크

본문 안에서 다른 페이지로 링크:
- 마크다운/HTML 변환이 없으므로 현재는 일반 텍스트만
- 본문 끝에 "관련 테스트" 섹션을 컴포넌트로 추가하는 패턴은 가능 (예: `BlogAmbiguousSituationshipPromo`)

## 이미지

- 대표 이미지 1개 권장 (`post.image`)
- 본문 inline 이미지는 현재 미지원 (MDX 도입 시 검토)
- 이미지 없는 글: `QUIZ_IMAGE_PENDING_SRC` fallback 자동

## 카테고리 / 태그

| 카테고리 | 용도 |
|---|---|
| `love` | 연애·썸·이별 |
| `personality` | 성격·MBTI |
| `tarot` | 타로 (현재 `/ko/today/` placeholder와 연관) |
| `fun` | 가벼운 재미 |
| `story` | 운영 노트·업데이트 |

| 태그 | 용도 |
|---|---|
| `가이드` | 기능·서비스 안내 |
| `에디터노트` | 운영 에디터 글 |
| `업데이트` | 서비스 업데이트 공지 |
| `팁` | 활용 팁 |

## 톤

[content-tone.md](./content-tone.md) 참고.

- 친근하고 가벼운 톤
- 의학·임상 단정 X
- "반드시", "무조건" 같은 단정 표현 자제
- 사용자를 평가하지 않음
- "그럴 수 있어" 공감 톤

## AdSense 심사를 위한 추가 점검

- [ ] 본문 600자 이상
- [ ] 명확한 주제 (제목 그대로의 내용)
- [ ] 표절·중복 X (다른 사이트 복사 절대 X)
- [ ] 차별·혐오·과장된 의학 표현 X
- [ ] 이미지 있음 (선택)
- [ ] 면책 톤 (필요 시): "본 내용은 일반 정보이며 진단·치료가 아닙니다"

## 새 글 발행 절차

1. `src/content/blog/koSamplePosts.ts`의 배열에 객체 추가
2. (선택) 대표 이미지 `public/images/blog/<...>.webp`
3. `public/sitemap.xml`에 `<url>` 등록 (priority 0.5, monthly)
4. `npm run build` 통과
5. `/ko/blog/`에서 카드 확인
6. `/ko/blog/<slug>/`에서 상세 확인

## 글 수정 / 제거

- 수정: 객체 필드 변경 후 빌드
- 임시 비공개: 객체를 주석 처리하거나 다른 배열로 이동 (현재 패턴 없음 → 직접 제거)
- 삭제 시 sitemap에서도 같이 제거

## 향후 개선

- MDX 도입 (마크다운 본문 직접 작성 가능, H2/H3/링크/이미지 지원)
- 글 검색 / 페이지네이션
- 카테고리 필터 강화
